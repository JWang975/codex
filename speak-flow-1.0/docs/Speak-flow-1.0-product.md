# Speak flow 1.0 产品文档

## 一句话定位

Speak flow 是一个 macOS AI 语音输入工具：用户按 Fn、打响指，或用手掌触发，说自然口语，系统自动转写、整理，并把文字插入当前光标位置。

## 产品愿景

键盘输入适合精确编辑，但不适合快速倾倒想法。Speak flow 的目标是把“说话”变成桌面系统里的第一等输入方式，让用户在任意 App 中直接说出想法、回复、任务和草稿。

1.0 不做复杂账号和订阅，不做硬件绑定，不做输入法。它专注于一个闭环：

```text
触发 → 录音 → 转写 → 整理 → 插入光标
```

## 目标用户

- 高频文字工作者：创业者、产品经理、研究员、运营、销售、创作者。
- 经常在多个 App 间切换的人：微信、飞书、Notion、浏览器、邮件、Cursor。
- 想减少键盘负担的人：长文本口述、快速回复、会议后整理、待办捕捉。

## 典型场景

### 1. 快速回复

用户正在微信或飞书输入框，按 Fn 后直接说：“帮我回复他说今天下午三点可以，我会提前十分钟到。”Speak flow 自动转成清晰文本并插入输入框。

价值：不用切 App，不用先录音再复制，直接在原位置完成输入。

### 2. 口语变书面表达

用户在 Notion 或文档里说一段想法，选择“润色”，系统把口语、停顿和重复整理成可阅读段落。

价值：降低从想法到文本的摩擦。

### 3. 多语言沟通

用户说中文，选择“翻译”，目标语言设为 English，系统直接把内容变成英文并插入邮件或聊天窗口。

价值：把翻译和输入合并为一个动作。

### 4. 临时任务捕捉

用户选择“待办”，口述：“明天上午提醒我整理会议纪要，再把报价单发给客户。”系统输出任务清单。

价值：自然语言直接进入行动项。

### 5. 无接触触发

用户双手不方便敲键盘时，打响指开始录音，再次响指停止；或者打开摄像头后用手掌触发。

价值：让语音输入像系统手势一样自然。

## 核心卖点

### 1. 输入入口极简

首页只告诉用户一件事：

```text
按 Fn，说话就输入
```

用户也可以用响指触发。手掌识别是可选视觉触发，不影响快捷键和响指。

### 2. 结果直接出现在光标处

转写完成后，Speak flow 会复制最终文本、激活原 App，并发送粘贴指令。用户不需要手动复制粘贴。

### 3. 默认原文，按需整理

默认“原文输入”追求速度；需要更正式表达时再切换“润色 / 翻译 / 待办”。

### 4. 触发方式解耦

- 快捷键：Fn 或自定义键盘快捷键。
- 响指：只依赖麦克风。
- 手掌：只在摄像头开启时生效。

关闭摄像头不会影响 Fn 和响指。

### 5. 云端转写 + 智能整理

1.0 推荐组合：

- ASR：Azure OpenAI Whisper
- LLM：DeepSeek

系统也保留本地 whisper.cpp 的后续扩展入口。

## 1.0 功能清单

### 首页

- 浅色极简界面。
- 显示状态：就绪 / 正在录音 / 正在转写 / 正在整理 / 已插入。
- 主按钮：开始语音输入 / 停止语音输入。
- 当前设置入口：`当前：原文输入 ›`。

### 当前设置页

- 输出方式：原文、润色、翻译、待办。
- 模式说明：
  - 原文：直接输入转写结果。
  - 润色：整理口语和错别字。
  - 翻译：翻译成目标语言。
  - 待办：提取任务清单。
- 状态列表：快捷键、响指、摄像头与手掌、转写服务、智能整理。

### 设置页

- 快捷键：支持 Fn、组合键、重置为 Left Control + Space。
- 响指：独立开关，只依赖麦克风。
- 摄像头与手掌：摄像头开关、手掌识别、摄像头选择。
- 转写服务：Azure OpenAI Whisper / OpenAI Whisper / 本地 whisper.cpp 入口。
- 智能整理：DeepSeek / OpenAI / Anthropic 兼容配置、模型、温度。

### 顶部菜单

- 开始/停止语音输入。
- 当前输出。
- 输出方式切换。
- 快捷键状态。
- 响指开关。
- 摄像头与手掌入口。
- 打开主面板、设置、版本、退出。

## 交互原则

- 首页只放行动，不放技术配置。
- 服务状态和设备状态收进当前设置和设置页。
- 错误提示用短中文，不暴露长堆栈。
- 录音/转写浮层小巧，不抢焦点。
- 所有配置保存后全局同步，顶部菜单和客户端一致。

## 产品边界

Speak flow 1.0 不包含：

- 账号系统
- 订阅系统
- 跨设备同步
- Windows 版本
- iOS 输入法
- 已签名/公证安装包

1.0 的重点是把 macOS 本地体验打磨到能稳定验证核心价值。

## PPT 建议结构

1. 标题：Speak flow 1.0 - 说话就输入
2. 痛点：键盘输入慢、跨 App 复制粘贴麻烦、口语到文本成本高
3. 产品定义：Fn / 响指 / 手掌触发的 AI 语音输入工具
4. 核心流程：触发 → 录音 → 转写 → 整理 → 插入
5. 典型场景：回复、写作、翻译、待办、无接触输入
6. 功能卖点：极简入口、自动插入、模式切换、触发解耦
7. 1.0 形态：浅色首页、当前设置、设置页、顶部菜单
8. 技术路线：Electron + React + Python helper + Swift helper + Azure Whisper + DeepSeek
9. 后续路线：签名公证、本地 ASR、个人词典、更多平台
